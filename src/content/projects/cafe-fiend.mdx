### Overview

A personal app for finding cafes without repeating the same searches. It builds on what Google Maps already does, but adds control over what shows up: filtering out places you've already seen, saving favorites, keeping a wishlist, and hiding spots you never want to revisit.

The project also doubles as a playground for <Anchor link="https://tanstack.com/start/latest" text="TanStack Start" variant="link" isExternal /> and some AI-assisted UI tooling, with plans to layer in AI search later.

### Key Features

1. **Flexible location input**  
  Use current location, search via autocomplete, or drag a map marker.
2. **Custom place states**  
  Favorites and wishlist (like Google Maps)
  Hidden places (excluded from results entirely)
3. **Smart filtering**  
  Filters by rating, review count, radius, and open status
  Deduplicates results against saved places
4. **Realtime data handling**  
  Cached queries and background updates using TanStack Query
  Optimistic updates for instant UI feedback
5. **Authentication**  
  Cookie-based sessions via Supabase
  Route-level protection for authenticated pages

### Implementation

The app is built with TanStack Start, mainly for its server functions and the ability to keep everything in TypeScript without splitting logic across API routes. Server functions handle data access directly, including authenticated queries against Supabase.

Example of a `createServerFn` for getting favorites.

```typescript
export const getFavorites = createServerFn({ method: "GET" }).handler(async () => {
  const supabase = getSupabaseServerClient();
  const { data: { user } } = await supabase.auth.getUser();
  
  if (!user) throw new Error("Unauthorized");
  
  return await supabase
    .from("saved")
    .select("*")
    .eq("user_id", user.id);
});
```

Authentication is managed through Supabase with cookie-based sessions. Protected routes live under a shared layout that checks for a user before loading, so anything inside that route tree requires login by default.

```typescript
// _authed.tsx - Layout route for protected pages
export const Route = createFileRoute("/_authed")({
  beforeLoad: ({ context }) => {
    if (!context.user) {
      throw redirect({ to: "/login" });
    }
  },
});
```

Location and discovery rely on Google Maps APIs. Autocomplete handles search input, while map interaction (including draggable markers) controls the coordinates used for fetching nearby cafes. This is the most expensive part of the app — API usage ramps up quickly.

User preferences are stored in a single Supabase table. Each place is tied to a user and a status (favorite, wishlist, or hidden). A unique constraint on (place_id, user_id) prevents duplicates, and upsert operations make switching states simple and atomic.

All remote data is wrapped in TanStack Query. Query keys include location and filters, so moving the map or changing preferences triggers fresh fetches automatically. Mutations (like favoriting a place) use optimistic updates, so the UI responds instantly before the server confirms the change.

```typescript
const { data, isLoading } = useQuery({
  queryKey: ["coffeeShops", lat, long, filters, favorites],
  queryFn: () => findNearbyCafes({ lat, long, filters }),
  enabled: !!(lat && long),
  refetchOnWindowFocus: false,
});
```

UI development leans on AI-generated components. Tools like <Anchor link="https://v0.dev" text="v0.dev" variant="link" isExternal /> were used to sketch layouts quickly, then refined with Tailwind CSS, Radix UI, and Lucide icons.
