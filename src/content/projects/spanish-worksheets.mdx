## Overview

A Django REST API deployed to Railway that automatically generates Spanish homework worksheets and delivers them by email every two days. A GitHub Actions CRON job authenticates with the API, triggers worksheet generation via DeepSeek's LLM, and sends the result through Mailgun — no manual intervention required.

Worksheets are scoped to work and technology contexts (debugging, pull requests, client negotiations) and cover four exercise sections: Past Tense, Present Tense, Future Tense, and Translation.

<hr />
## Key Features

- **Automated delivery** — GitHub Actions CRON runs every two days, wakes the Railway service, and calls the generate endpoint
- **LLM-generated exercises** — DeepSeek (`deepseek-chat`) produces 7 exercises per section, emphasising irregular verbs and real work contexts
- **Topic rotation** — 10 theme pools (bugs, deploys, client negotiations, etc.) rotate automatically via a `Config` key-value store
- **Async generation** — Worksheet creation is queued via Redis + RQ so the API responds immediately with a job ID
- **Duplicate prevention** — Content is SHA-256 hashed before saving; duplicate worksheets are rejected
- **Multi-recipient email** — Each user can add extra email recipients; all receive the same worksheet via Mailgun

## Implementation

### Models

**User** — Custom `AbstractBaseUser` using email as the login field. Tracks `active` status and `next_delivery` date.

**Worksheet** — One record per user (replaced on each generation). Stores the raw `content` JSON, a `content_hash` for deduplication, the `topics` covered, and the `themes` used.

**Config** — Simple key-value table. Currently holds `topic_index`, an integer that advances on each generation to cycle through the theme pools.

**UserRecipient** — Additional email addresses per user. Enforces uniqueness on `(user, email)`.

### Async Job Queue

`/api/worksheet/generate-worksheet/` enqueues a Redis RQ job (`generate_worksheet_job`) and returns a `job_id` immediately. The job handles LLM generation, deduplication, database persistence, and email delivery. Job status can be polled at `/api/worksheet/worksheet-job-status/<job_id>/`.

### Content Generation

1. Resolve themes — use the topic rotator or fall back to caller-supplied themes
2. Build a system + user prompt and call the DeepSeek API
3. Extract JSON from the response (tries direct parse → markdown code block → raw object extraction)
4. Attempt one-shot LLM repair if extraction fails
5. Validate structure — exactly 4 sections, 7 exercises each, with non-empty `prompt` and `answer` fields
6. Hash the content and reject if a duplicate already exists for the user

### Endpoints

| Endpoint | Method | Auth | Purpose |
|---|---|---|---|
| `/api/token/` | POST | ❌ | Exchange email + password for a token |
| `/api/worksheet/generate-content/` | POST | ✅ | Generate worksheet content only (no save, no email) |
| `/api/worksheet/generate-worksheet/` | POST | ✅ | Queue full generation + email, return `job_id` |
| `/api/worksheet/send-worksheet-email/` | POST | ✅ | Resend the user's latest worksheet by email |
| `/api/worksheet/latest/` | GET | ✅ | Fetch the user's most recent worksheet |
| `/api/worksheet/worksheet-job-status/<job_id>/` | GET | ✅ | Poll the status of a queued generation job |

### Email Delivery

Mailgun (EU endpoint) sends both HTML and plain-text versions to the user's primary address and any additional `UserRecipient` addresses. The four exercise sections are shuffled before rendering for variety.

### GitHub Action

The workflow runs on `cron: '0 14 */2 * *'` (every two days at 14:00 UTC) and can be triggered manually.

1. Poll the Railway service until it responds (up to 24 attempts)
2. Request an auth token with retries
3. Call `/api/worksheet/generate-worksheet/`
