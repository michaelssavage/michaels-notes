### Overview

A full-stack TypeScript application that automatically creates a Spotify playlist and populates it with new releases from your saved artists. Built with React and Express in a monorepo architecture, it runs a daily sync via GitHub Actions and exposes a dashboard for managing tracked artists, browsing liked songs, and reviewing playlist updates.


### Key Features

#### Playlist automation

A GitHub Actions workflow fires daily at midnight UTC, calling a JWT-protected `/api/trigger-schedule` endpoint on the server. The scheduler fetches every user's tracked artists, checks the Spotify API for albums released that day, and appends any new tracks to the user's "New Music Releases" playlist. A manual trigger with a custom `fromDate` param is also supported for backfills.

#### Saved artists

Users can search the Spotify catalogue and pin artists for tracking. The table is built with TanStack React Table and supports column sorting and fuzzy text filtering. Clicking an artist opens a side panel with follower count, genres, and a direct Spotify link.

#### Playlist updates

Displays the tracks currently in the "New Music Releases" playlist with album art, release date, and a deep link back to Spotify. Admins can manually trigger a sync from the UI.

#### Liked songs

Renders all of the user's liked tracks with infinite scrolling powered by the Intersection Observer API, so the full library loads progressively without pagination controls.

### Implementation

#### Backend architecture

<Anchor link="https://expressjs.com/" text="Express.js" variant="link" isExternal /> handles all API routes, serves the compiled frontend as static assets, and manages Spotify OAuth. The codebase follows a Clean Architecture split across four layers — Router, Controller, Service, and Repository — with <Anchor link="https://en.wikipedia.org/wiki/Dependency_injection" text="dependency injection" variant="link" isExternal /> wiring them together via factory functions. A `ServiceContainer` composes the dependencies at startup, making each layer independently testable.

> Router → Controller → Service → Repository → MongoDB

The `SpotifyService` is the core domain layer. It handles OAuth token exchange and refresh, manages the tracked-artists list, and owns the full release-sync pipeline: fetching albums per artist, filtering by release date, creating the playlist if it doesn't exist yet, and appending tracks. `p-limit` caps concurrent Spotify API calls at five to stay within rate limits when a user tracks hundreds of artists.

The `SchedulerService` wraps job execution with safeguards — it skips a run if the previous one is still in progress or ran within the last 20 hours, unless the trigger is flagged as manual. Each execution is logged to MongoDB with its status, duration, and any error.

#### Frontend

<Anchor link="https://vite.dev/" text="Vite" variant="link" isExternal /> builds the frontend in 4-5 seconds and outputs static assets that Express serves directly, keeping the deployment a single process. The UI is TypeScript and React, styled with Emotion CSS-in-JS.

<Picture src="/project/nmr_tanstack.png" alt="TanStack apps" />

<Anchor link="https://tanstack.com/router" text="TanStack Router" variant="link" isExternal /> handles file-based routing with built-in auth guards (`requireAuth` / `requireUnauth`) that run before each route renders. <Anchor link="https://tanstack.com/query" text="TanStack Query" variant="link" isExternal /> manages all server state — caching, background refetches, and mutations — while <Anchor link="https://zustand.pmnd.rs/" text="Zustand" variant="link" isExternal /> holds lightweight client-only state like the authenticated user and the saved artists list.

#### Authentication

Spotify OAuth 2.0 uses the authorisation code flow. After consent, the server exchanges the code for tokens and redirects to the frontend callback route with the tokens as query params. Tokens are stored in secure, HTTP-only cookies (1-hour access, 7-day refresh). The client sets up a 4-minute interval to silently refresh the access token before it expires.

#### Deployment

The app runs on <Anchor link="https://render.com/" text="Render" variant="link" isExternal /> as a single service — the Express server handles both the API and static file serving. The daily playlist sync is driven by a GitHub Actions scheduled workflow that POSTs to the `/api/trigger-schedule` endpoint using a JWT stored as a GitHub Secret.
