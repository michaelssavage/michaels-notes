### Overview

A final year project focused on building a desktop emotion classifier that analyzes text a user reads online. The app is a Python GUI built with <Anchor link="https://www.pythonguis.com/pyqt5-tutorial/" text="PyQt5" variant="link" isExternal /> and <Anchor link="https://doc.qt.io/qt-6/index.html" text="Qt Designer" variant="link" isExternal />, combining web scraping, NLP, and machine learning to classify emotional tone in real time.

### Key Features

1. **Live text analysis**  
  Extracts text from web pages the user is viewing
  Processes and classifies content in near real-time
2. **Emotion classification**  
  Classifies six emotions: anger, fear, joy, surprise, happiness, sadness
  Moves beyond basic sentiment (positive/negative/neutral)
3. **Balanced model training**  
  Ensures equal representation of emotions during training
  Reduces bias in classification results
4. **Data visualization**  
  Displays results using charts and histograms
  Updates dynamically as new data is processed
5. **Desktop GUI**  
  Built with PyQt5 and Qt Designer
  Uses thread pools to keep the interface responsive

### Implementation

Text extraction starts by scraping content from pages the user visits. <Anchor link="https://beautiful-soup-4.readthedocs.io/en/latest/" text="BeautifulSoup" variant="link" isExternal /> is used to parse page content, while <Anchor link="https://github.com/scrapinghub/splash" text="Splash" variant="link" isExternal /> helps render JavaScript-heavy sites before extraction. Browser history, stored in SQLite databases, provides a source of URLs to process.

The NLP pipeline begins with basic sentiment analysis before moving into deeper emotion classification. Text is parsed and lemmatized using spaCy, then processed with NLTK for working with emotion datasets.

The classifier is trained locally using <Anchor link="https://scikit-learn.org/stable/" text="scikit-learn" variant="link" isExternal />. To avoid skewed predictions, pandas is used to balance the dataset so each emotion class is equally represented during training.

The GUI layer is handled by PyQt5, with background processing managed through thread pools so classification runs without freezing the interface. Results are visualized using Plotly, updating charts in real time as new text is analyzed.
