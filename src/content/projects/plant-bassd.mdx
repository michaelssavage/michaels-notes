### Overview

<Picture src="/project/plantbassd-landing.png" alt="Home page of the app" />

Plantbassd.com is a personal project built with <Anchor link="https://nextjs.org/" text="Next.js" variant="link" isExternal /> and TypeScript. It started as a simpler setup using JavaScript, CSS, and markdown-based blog posts, then evolved steadily over three years through continuous refactoring.

The project doubled as a learning space. Changes weren't planned upfront, and they came from experimenting, hitting limitations, and fixing them. A running list of improvements lived in Trello and drove most updates.

### Key Features

1. **Typed codebase**  
  Migrated from JavaScript to TypeScript to reduce bugs and improve reliability.

2. **MDX-powered blog**  
  Replaced markdown with MDX to allow JSX components inside posts.  

3. **Image handling and optimization**  
  Initially relied on Vercel's free-tier image optimization (WebP, SSG performance gains). Later moved assets to <Anchor link="https://cloudinary.com/" text="Cloudinary" variant="link" isExternal /> to avoid rebuilds and reduce repo size.

4. **Styling evolution**  
  Started with Bootstrap 5 + CSS. Transitioned to SCSS for better structure, reuse, and maintainability (mixins, functions).

5. **Design and media**  
  Custom article cover images created using Adobe Photoshop and Illustrator.  Event visuals produced with Adobe Premiere Pro (including a 20-minute themed video compiled from ~200 films).

### Implementation

The development of Plantbassd.com followed a gradual, incremental approach rather than a full rewrite. Each change was introduced as new challenges or limitations emerged, allowing the project to evolve organically over time.

The content system underwent a significant upgrade when the limitations of markdown became apparent. Markdown's rigid formatting prevented embedding interactive or UI components, so the project transitioned to MDX. This allowed JSX components to be used directly within blog posts, providing greater flexibility and functionality.

Image handling was another area of focus. Originally stored within the repository, images were moved to Cloudinary. This decoupling allowed media to be updated or swapped without triggering a full rebuild of the site on GitHub, improving maintainability and deployment efficiency.

Performance was addressed through a combination of Static Site Generation (SSG) and Vercel's built-in optimizations, including WebP image handling. Offloading images externally further reduced the load on the platform and helped stay within service limits.

Finally, the repository structure was revisited. The early layout had not been designed for long-term scalability, so a reorganization was carried out to make the project easier to navigate, extend, and maintain as it grew.
